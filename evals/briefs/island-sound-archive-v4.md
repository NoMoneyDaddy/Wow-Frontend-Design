# 固定測試題 B：島嶼聲音檔案館

建立繁體中文、面向一般讀者與研究者的島嶼聲音檔案探索網站。讀者可能在手機上快速尋找一段口述記憶，也可能在桌機閱讀逐字節錄與採集脈絡。這是內容先行的 editorial archive，不是 dashboard、串流平台首頁或通用行銷模板。

## 情境與內容

- 至少呈現八筆唯一錄音，混合島嶼／地點、講者、語言或腔調、年代、長度、主題、授權與可聆聽狀態。
- 清楚區分題名、講者、編輯摘要、採集資訊與權利狀態；不可把所有內容塞成同尺寸卡片。
- 需要一段有編輯意圖的繁中直式排版，作為地名、檔案索引或節錄，並與橫排資訊建立清楚閱讀順序。中文字保持直立，不可旋轉整個容器冒充直排。
- 視覺可參考田野筆記、錄音帶標籤、潮線、航海座標與檔案盒，但內容、來源與閱讀節奏優先。避免 KPI 牆、SaaS 側欄、付費方案、通用漸層 hero、假播放器與裝飾性卡牆。

## 響應式與媒體 fallback

- Desktop 讓直排與橫排形成有意義的編輯構圖。Mobile 必須提供同內容的較短水平等價版本，保留探索入口；不可把長直排硬塞進窄螢幕。
- 首屏需看得到品牌、目前檔案脈絡及下一步。導覽預設關閉時不得蓋住內容。
- 需容納較長的繁中題名、雙語地名、完整權利說明與逐字節錄。短按鈕、年份、品牌及狀態不得意外換行、裁切、碰撞或溢位。
- 不提供真實音訊檔。若呈現播放控制，必須誠實標示為預覽／不可播放狀態，且文字內容不依賴音訊才能理解。

## 評測掛鉤

- 使用 `lang="zh-Hant"`。
- 直式區域使用 `data-eval="vertical-type"` 與實際 `writing-mode: vertical-rl`。
- 每筆錄音容器使用 `data-eval="record"` 與唯一 `data-record-id`。
- 全域導覽使用 `data-eval="global-nav"`；若有 mobile 導覽按鈕，使用 `data-eval="mobile-nav-toggle"`。固定全域主要動作若存在，使用 `data-eval="global-action"`。

## 限制

- 不使用外部套件、網路資產或 network request。
- 沒有獨立瀏覽器證據時，不得宣稱音訊、互動、可用性、授權狀態或 WCAG 已通過。
