# 固定測試題 A：山域救援任務協調台

建立繁體中文的登入後產品介面，供山域救援指揮員監看與比較任務。這是高密度工作台，不是 landing page、形象網站或行銷頁。

## 使用者與視覺目標

- 使用者同時監看多個責任區、通訊狀態、任務急迫度、時限與隊伍負荷。
- 畫面應精準、克制、高可掃讀，可從地形、座標、無線電與耐候現場標記取得產品語彙。
- 危險色只表示真正危險或超時。急迫度、任務階段與期限是獨立語意，不可互相染色。
- 避免大型 hero、行銷 CTA、pricing、testimonials、玻璃擬態、霓虹漸層、滿版紅色警報與裝飾性卡牆。

## 固定內容與外觀檢查點

- 使用 `lang="zh-Hant"`，呈現至少八筆唯一任務，資料涵蓋不同責任區、急迫度、階段、時限與隊伍。
- 具備產品 shell、主要導覽、頁面標題、摘要及主工作區。摘要不能取代任務列表。
- Desktop 利用寬度做橫向比較；mobile 重新構成任務優先 inbox，壓縮導覽與次要資訊，不可只是縮小或單純堆疊 desktop。
- 短按鈕與狀態文字不得意外換行、裁切或溢位。
- 每筆任務容器使用 `data-eval="record"` 及唯一 `data-record-id`；其急迫度、階段與時限分別使用 `record-priority`、`record-status`、`record-due`，供獨立視覺稽核比較語意顏色。
- 導覽使用 `data-eval="global-nav"`；若 mobile 有導覽按鈕，使用 `data-eval="mobile-nav-toggle"`。
- 所有資料只是靜態 demo。外觀驗證不要求後端、登入、遠端同步或完整產品流程。

## 限制

- 不使用外部套件、網路資產或 network request。
- 沒有瀏覽器證據時，不得宣稱畫面、互動、可用性或 WCAG 已通過。
